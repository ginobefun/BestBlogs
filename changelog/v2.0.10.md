## 中文

### ✨ 新功能

**兴趣标签精准化：per-tier 数量上限**

每条内容的兴趣标签现在按分层严格限制：领域标签最多 1 个、主题标签最多 5 个、形式标签最多 1 个、实体标签最多 3 个（合计不超过 10 个）。打标质量更高，推荐相关性更精准。

### 🔧 体验优化

**翻译工具栏重组与视觉升级**

- 桌面端工具栏重组：Copilot 与翻译按钮直接可见，分享等次要操作收入「更多」菜单
- 移动端底部工具栏精简，核心操作（收藏 / Copilot / 翻译 / 分享）一行显示
- 翻译中、完成、失败等状态视觉统一升级，深色模式对比度同步修正

### 🐛 修复

**订阅源探索：「仅展示未关注」不再为空**（#303）

「添加新来源」页面开启「仅展示未关注」后，列表显示为空的 Bug 已修复。根本原因：过滤逻辑此前在前端执行，关注数量较多时过滤后结果为空；现已改为服务端 MongoDB NOT IN 过滤，`totalCount` 也随之准确。

**订阅源探索：语言筛选恢复生效**（#302）

访问 `/explore/sources?language=en` 或 `?language=zh` 时，语言筛选现在正确生效，仅返回对应语言的订阅源。此前 `language` 参数未传入 MongoDB 查询，导致筛选完全无效；同步修复短码（`en`/`zh`）→ 存储格式（`en_US`/`zh_CN`）的转换逻辑。

**Vimeo 视频嵌入恢复正常播放**（#301）

文章中嵌入的 Vimeo 视频播放失败的问题已修复，点击视频可正常播放。

## English

### ✨ New Features

**Interest Tag Precision: Per-Tier Limits**

Interest tags are now strictly capped per tier: domain ≤1, topic ≤5, format ≤1, entity ≤3 (max 10 total per article). This results in higher-quality tagging and more relevant recommendations.

### 🔧 Improvements

**Translation Toolbar Redesign & Visual Upgrade**

- Desktop toolbar reorganized: Copilot and Translate buttons are always visible; secondary actions (Share, etc.) moved to a "More" menu
- Mobile bottom bar streamlined: core actions (Bookmark / Copilot / Translate / Share) shown in one row
- Translation states (loading, complete, error) get a unified visual refresh with improved dark-mode contrast

### 🐛 Fixes

**Source Discovery: "Unsubscribed Only" No Longer Returns Empty List**（#303）

Fixed a bug where enabling "Show Unsubscribed Only" in the Add Sources page returned an empty list. Root cause: client-side filtering after backend pagination left zero results when the user had many subscriptions. The filter is now applied server-side via MongoDB NOT IN query, and `totalCount` is accurate.

**Source Discovery: Language Filter Now Works**（#302）

Visiting `/explore/sources?language=en` or `?language=zh` now correctly filters sources by language. The `language` parameter was previously ignored in the MongoDB query. Short codes (`en`/`zh`) are now properly normalized to storage format (`en_US`/`zh_CN`).

**Vimeo Videos Now Play Correctly**（#301）

Fixed Vimeo video playback failure in articles.
