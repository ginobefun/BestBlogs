## 中文

### ✨ 新功能

**开放能力首发：让 BestBlogs 能被你的工作流调用**

BestBlogs 第一次把平台能力对外开放。你可以在命令行、脚本、或 AI 助手（Claude Code、Cursor 等）里直接用：今日最值得读、个性化推荐、关注流、搜索、精选周刊、深度阅读、收藏与笔记。

- **命令行工具 `@bestblogs/cli`**：一行 `npm install -g @bestblogs/cli` 装好，`bestblogs discover today` 就能看到今天最值得读的内容；配合 `--json` 参数可直接接入自动化脚本
- **Agent Skills 一键接入**：支持 SKILL.md 约定的智能体（Claude Code、Cursor 等）零配置调用，问一句"今天 BestBlogs 有什么值得读的？"即可
- **每条推荐都会解释自己**：告诉你它来自 Pro 个性化早报 / 公共精选 / For You / 关注源中的哪一条，并给出一句话说明"为什么推这条"——内容平台应该是透明的
- **开发者 API**：完整的 OpenAPI 3.1 规范文档，可导入 Postman / 函数调用 schema 生成器；所有端点支持按量限流和 API Key 管理

### 🔧 体验优化

**早报卡片密度切换**
- 早报阅读页新增「紧凑 / 舒适」切换按钮，两种密度自由选——想一屏看到更多就切紧凑，想让图片和摘要透气就切舒适
- 选择会自动记住，下次回来保持你的偏好

**全局播放器统一支持播客**
- 打开播客后再切到别的页面，底部播放器会随你继续播放，不再出现「底部多出一条 bar」的错位

### 🐛 修复

**移动端**
- 文章中的宽图在手机上不再溢出屏幕，固定像素的图片也能正确缩放
- 分页按钮在窄屏改为紧凑的「‹ 当前页 / 总页 ›」样式，不再挤满整行
- 打开 AI 伴读时弹出键盘，输入框不再被键盘遮挡；选中文本时的浮动工具栏不再覆盖 AI 伴读界面
- 文章底部的快捷按钮栏现在可以横向滑动，右侧渐变遮罩提示还有更多按钮

**收藏夹 / 内容广场 / 早报**
- 收藏夹页面在后端临时波动时不再整页报错，现在会自动重试并显示「加载失败 · 重新加载」按钮
- 内容广场刷新页后，第一次切换分类或筛选条件立即生效（之前需要切第二次才刷新）
- 早报卡片的「点赞」按钮不再挡住标题，移动端改为卡片下方独立操作区

**视频 / AI 伴读**
- 桌面端打开含 Vimeo 视频的文章不再黑屏
- AI 伴读对话出错时，消息气泡会显示红色「重试」按钮，点一下即可重新提问

## English

### ✨ New Features

**Open Capabilities Launch — Make BestBlogs Part of Your Workflow**

For the first time, BestBlogs opens its core capabilities to the outside world. Use today's picks, personalized feeds, following streams, search, newsletters, deep reading, bookmarks and notes directly from your terminal, scripts, or AI assistants (Claude Code, Cursor, and more).

- **`@bestblogs/cli` command-line tool**: Install with `npm install -g @bestblogs/cli`, then `bestblogs discover today` shows today's best reads. Pair with `--json` to pipe into any automation
- **One-click Agent Skills**: Drop-in support for any SKILL.md-compatible agent (Claude Code, Cursor...). Just ask "what's worth reading on BestBlogs today?"
- **Every recommendation explains itself**: See whether it came from your Pro personalized brief, the public pick, For You, or your following stream — with a one-line reason. Content platforms should be transparent
- **Developer API**: Full OpenAPI 3.1 spec, importable into Postman or LLM function-calling schema generators. All endpoints come with rate-limiting and API key management

### 🔧 Improvements

**Density Toggle for Daily Brief Cards**
- A new "Compact / Comfortable" toggle lets you choose between seeing more on one screen or giving images and summaries room to breathe
- Your choice is remembered across sessions

**Unified Podcast Playback**
- Podcasts now play through the global player. Switching pages mid-playback no longer leaves an extra player bar floating at the bottom

### 🐛 Fixes

**Mobile**
- Wide article images no longer overflow the screen; images with fixed pixel widths scale correctly
- Pagination compacts to "‹ current / total ›" on narrow screens, no longer wrapping to multiple lines
- Keyboard no longer hides the input when AI Copilot is open; the text selection toolbar no longer covers the Copilot panel
- The bottom action bar on article pages is now horizontally scrollable, with a right-edge gradient hinting at more actions

**Library / Content Hub / Daily Brief**
- The bookmark page no longer errors out on transient backend blips — it now retries automatically and shows a "Retry" button if needed
- In the content hub, the first filter change after a page reload now takes effect immediately (previously required a second toggle)
- The "like" button on daily brief cards no longer covers the title on mobile — it moved to a dedicated action row below the card

**Videos / AI Copilot**
- Vimeo videos in articles now play correctly on desktop
- When an AI Copilot message fails, the bubble now shows a red "Retry" button to resend the question
