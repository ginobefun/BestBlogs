## 中文

### ✨ 新功能

**Daily Review 开始发邮件了（Pro 内测）**
- Pro 内测用户每天会自动收到一封中英双语的当日 Daily Review，标题包含日期与重点
- 在 Gmail / Outlook / QQ 邮箱 / Apple Mail 都能正确显示，按钮和排版不再错乱
- 邮件底部一键退订（中英双语反馈页），退订后会自动跳过后续投递
- 投递通道有备份保障，主线异常时系统会自动切换，不影响"早上一定能收到"

**让 BestBlogs 出现在 ChatGPT / Perplexity / Claude.ai 的搜索结果里**
- 显式放行检索类 AI 助手抓取，让用户在它们里面问相关问题时能看到 BestBlogs 的内容
- 显式屏蔽用于训练大模型的爬虫，保护内容来源方与创作者的劳动
- 后台、登录、订阅管理等内部页面继续保持屏蔽

**管理后台核心数据看板**
- 后台新增「核心数据」一屏概览：日活、留存、Pro 转化、关键事件趋势
- 内容、Pro、功能使用等模块各有独立的下钻页

### 🔧 体验优化

**Pro 早报中英文严格按 URL 分流**
- 打开 `/zh/reading/brief` 始终是中文、`/en/reading/brief` 始终是英文
- 不再受到部分移动浏览器或运营商代理改写语言信息的影响
- 桌面端 hover 卡片时弹出的摘要与卡片显示的文字保持一致

**早报内联图统一为 16:9 横图**
- 早报详情页（精讲、速览、扩展阅读）所有内联图都统一渲染为 **16:9 横图**，铺满栏宽、居中裁剪
- 圆角与上下间距一致，告别「小方头像 + 大方块 + 宽横图」混排
- 仅作用于早报，文章正文 / 推文 / 播客 shownotes / 周刊不受影响

**内容广场移动端补齐类型筛选**
- `/explore` 移动端新增类型切换条（全部 / 文章 / 播客 / 视频 / 推文），与桌面端共享同一筛选状态
- 卡片元信息行新增类型小标签（图标 + 文字），扫一眼就能区分内容形式
- 类型切换条与下方筛选预设视觉层级清晰，不再"挤成一堆 chips"

**列表与搜索明显提速**
- 内容广场、阅读历史等首屏接口都比之前快了一截
- 搜索结果只返回必要字段，加载更快、流量更小
- 个性化开关请求失败时使用本地缓存兜底，不再因接口慢而拖慢首屏

### 🐛 修复

- Pro 早报在部分移动浏览器（运营商会改写语言信息）下中英文显示错乱 — 改为按 URL 严格判断
- 早报内联图尺寸混乱影响阅读节奏 — 统一为 16:9 横图
- 内容广场移动端缺类型筛选入口、且卡片无法一眼分辨内容形式 — 补齐筛选条与类型标签
- 后台 `/analytics`、`/dashboard` 部分进度条与图表轴在暗黑模式下对比度不足 — 改用语义色变量重做

## English

### ✨ New Features

**Daily Review Now Lands in Your Inbox (Pro Closed Beta)**
- Pro beta users automatically receive a bilingual (zh + en) Daily Review email each day, with the date and key highlights right in the subject line
- Renders correctly across Gmail, Outlook, QQ Mail, and Apple Mail — no more broken buttons or layouts
- One-click unsubscribe with a bilingual feedback page; once you opt out, future emails are skipped automatically
- Delivery has a backup channel — if the primary path has an issue, the system fails over so your morning brief still arrives on time

**BestBlogs Now Reachable from ChatGPT / Perplexity / Claude.ai**
- Explicitly allows retrieval-class AI assistants to index BestBlogs, so when users ask them related questions, BestBlogs content can show up in answers
- Explicitly blocks crawlers used for model training, protecting content sources and creators
- Internal, sign-in, and subscription management pages remain blocked

**Admin Core Analytics Dashboard**
- New "Core Analytics" panel in the admin console: DAU, retention, Pro conversion, and key event trends in one view
- Drilldown pages for content ops, Pro subscription, feature usage, and more

### 🔧 Improvements

**Pro Daily Brief — Locale Strictly Follows the URL**
- `/zh/reading/brief` always renders in Chinese; `/en/reading/brief` always in English
- No longer affected by mobile browsers or proxies that rewrite the language hint
- The PC hover tooltip on featured cards now matches the visible summary text exactly

**Daily Brief Inline Images Normalized to 16:9**
- All inline images in the daily brief detail view (deep dives + headlines + further reading) render as **16:9 landscape, full column width, center-cropped**
- Consistent rounded corners and spacing — no more mixed "tiny avatar + giant square + wide banner" layouts
- Scope is strictly the brief; article body, tweets, podcast shownotes, and the weekly digest are unaffected

**Mobile Type Filter on the Explore Feed**
- `/explore` adds a mobile type switcher (All / Article / Podcast / Video / Tweet) that shares the same URL state as the desktop tabs
- Cards now show a small type badge (icon + label) at the start of the meta row, so you can tell formats apart at a glance
- The type switcher is now visually distinct from the quick-filter chips below — no more "wall of chips"

**Lists and Search Are Noticeably Faster**
- The Explore feed and reading history feel snappier on first paint
- Search results return only the fields the page needs, so they load faster and use less bandwidth
- When the feature-flag fetch is slow or fails, the app falls back to a local cache instead of stalling the first paint

### 🐛 Fixes

- The Pro Daily Brief route relied on the language hint, causing some mobile browsers behind proxies to show Chinese to English readers (and vice versa) — now decided strictly by the URL
- Inconsistent inline image sizes in the brief (tiny squares mixed with big banners) hurt readability — renderer now normalizes them to 16:9 landscape
- The Explore feed had no mobile type filter and cards lacked an explicit type indicator — added a mobile switcher plus type badges
- Some progress bars and chart axes in admin `/analytics` and `/dashboard` had insufficient contrast in dark mode — migrated to semantic color tokens
