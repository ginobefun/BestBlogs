## 中文

### ✨ 新功能

**中英双语公共早报**
- 公共早报现在为中文和英文用户分别生成独立版本，内容候选池、推荐逻辑均相互独立
- 中文站访问者默认看到中文早报，英文站（/en）访问者默认看到英文早报
- RSS 订阅链接（`/zh/feeds/rss/daily-brief` 和 `/en/feeds/rss/daily-brief`）也分别返回对应语言版本

**早报精选内容折叠 + 精讲徽章**
- 早报内容区域默认展示精选的 10 条文章（评分和匹配度最高的 Top 10）
- 其余"扩展阅读"默认折叠，点击"查看更多 N 条扩展阅读"即可展开，避免信息过载
- 入选深度分析的文章在卡片左上角显示金色「精讲」徽章，方便快速识别

**公共早报登录后可点赞**
- 已登录的非 Pro 用户也可以对公共早报的精选文章点赞或点踩，不再强制要求 Pro 订阅
- 点赞状态持久保存，刷新页面不消失

**推荐理由 Markdown 渲染**
- 早报精选文章的个性化推荐理由支持 `**粗体**` 格式渲染，重点词以加粗突出

### 🛠 管理后台

**兴趣标签维护接口（4 个新增 Admin API）**
- 分页查询已完成分析但未打标的内容列表，优先展示高质量内容
- 为单条内容写入或覆盖标签编码（manual 标签优先级高于规则标签）
- 覆盖订阅源的结构化标签编码
- 分页查询兴趣发生漂移的用户列表，供重跑推荐

**Admin API 支持 X-API-KEY 鉴权**
- 管理后台 API 新增 X-API-KEY 头鉴权方式，方便脚本和 CI 自动化调用，无需浏览器会话
- JWT Bearer Token 和 X-API-KEY 两种方式任一满足均可通过校验

### 🐛 Bug 修复

**早报反馈按钮键盘焦点丢失**
- 之前：点赞/点踩按钮点击后焦点意外丢失，键盘用户无法继续操作
- 现在：点击反馈按钮后焦点正确保持，无障碍体验改善

**早报"功能关闭"状态闪烁**
- 之前：功能开关数据加载完成前会短暂显示"功能已关闭"，视觉跳变
- 现在：等待开关确认加载完成后才渲染关闭状态，页面过渡平滑

**早报加载状态缺少第三阶段**
- 之前：当天早报不存在但最近日期搜索还在进行中时，加载 spinner 提前消失
- 现在：三个阶段（flags 未加载 / 早报未加载 / 最近日期搜索中）均保持 loading 状态

**订阅源发现页已关注状态未同步**
- 之前：通过发现页关注或取消关注来源后，已关注状态没有实时同步到列表
- 现在：点击关注/取消关注后立即更新列表中的状态显示

**PostHog 事件上报被 CSP 拦截**
- 之前：Content Security Policy 未放行 PostHog 上报域名，导致用户行为事件静默丢失
- 现在：CSP 同时放行 US 和 EU 区 PostHog 域名（`us.i.posthog.com` / `eu.i.posthog.com`）

## English

### ✨ New Features

**Bilingual Public Daily Brief**
- The public daily brief now generates independent Chinese and English editions with separate candidate pools and recommendation logic
- Chinese-locale visitors see the Chinese brief by default; English-locale visitors (`/en`) see the English edition
- RSS feeds (`/zh/feeds/rss/daily-brief` and `/en/feeds/rss/daily-brief`) each return the corresponding language

**Featured Content Collapse + Deep-Dive Badge**
- The brief content section now shows only the top 10 featured articles by default (highest score and match quality)
- Remaining "extended reads" are collapsed; click "Show N more extended reads" to expand — reduces information overload
- Articles selected for deep-dive analysis display a gold "Deep Dive" badge in the top-left corner for quick identification

**Logged-in Users Can Rate Public Brief Articles**
- Non-Pro logged-in users can now thumbs-up or thumbs-down articles in the public daily brief — no Pro subscription required
- Rating state is persisted and survives page refresh

**Match Reason Markdown Rendering**
- Personalized match reasons in the brief now render `**bold**` formatting, making key highlights stand out

### 🛠 Admin

**Interest Tag Maintenance APIs (4 New Admin Endpoints)**
- Paginated list of analyzed-but-untagged content, prioritizing high-quality items
- Write or overwrite tag codes for a single content item (manual tags override rule-based tags)
- Overwrite structured tag codes for a subscription source
- Paginated list of users with interest drift detected, for recommendation re-runs

**X-API-KEY Authentication for Admin APIs**
- Admin APIs now accept X-API-KEY header authentication alongside JWT Bearer Token
- Either method satisfies authentication — makes script and CI automation easier without a browser session

### 🐛 Bug Fixes

**Brief Feedback Button Lost Keyboard Focus**
- Before: Clicking thumbs-up/down caused keyboard focus to disappear, blocking keyboard navigation
- After: Focus is correctly retained after clicking feedback buttons

**"Feature Disabled" Flash in Daily Brief**
- Before: The "feature disabled" state appeared briefly before feature flags finished loading, causing a visual flicker
- After: Disabled state only renders once flags are confirmed loaded

**Brief Loading State Missing Third Phase**
- Before: Loading spinner disappeared prematurely when today's brief didn't exist but nearest-date search was still in progress
- After: All three loading phases (flags / brief / date search) correctly maintain the loading state

**Source Discovery Subscription Status Out of Sync**
- Before: Follow/unfollow actions in the discovery page didn't immediately update the subscribed status in the list
- After: Status updates instantly on follow or unfollow

**PostHog Events Blocked by CSP**
- Before: Content Security Policy didn't allowlist PostHog reporting domains, silently dropping user analytics events
- After: CSP now allowlists both US and EU PostHog domains (`us.i.posthog.com` / `eu.i.posthog.com`)
