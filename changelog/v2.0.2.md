## 中文

### ✨ 新功能

**早报音频后台持续播放**
- 在早报页开始收听后，切换到文章、发现页或任意其他页面，音频不再中断
- 屏幕底部出现迷你播放条，随时查看进度、暂停继续、调整倍速
- 返回早报页自动恢复全功能播放器

**早报图文双视图**
- 新增「图文模式」，可以像读文章一样阅读早报内容
- 切换到图文模式后，音频在后台继续播放，边听边读两不误

**公共早报 RSS 订阅**
- 公共早报现在可以通过 RSS 订阅，在 Inoreader、NetNewsWire 等 RSS 阅读器中接收每日推送
- 附带完整音频信息，支持 Overcast、Pocket Casts 等播客 App 直接收听
- 订阅地址：`https://bestblogs.dev/zh/feeds/rss/daily-brief`（中文）

### 🐛 Bug 修复

**新开通 Pro 用户首次进早报的提示优化**
- 之前：当天在早上 7 点之后才开通 Pro，进入早报页会显示「正在为你准备」，暗示即将就绪，但实际要等到第二天才有早报
- 现在：首次订阅时明确显示「你的第一期早报明天就来」；已有历史早报的用户则显示「今天的早报还未就绪」并附上最近一期链接

**公共早报精选文章显示不完整**
- 之前：每条精选内容只显示标题，来源、类型、评分均缺失
- 现在：完整展示标题、来源名称、内容类型和质量评分

**字幕跟随时整页乱跳**
- 之前：开启字幕跟随后，浏览下方精选内容时整个页面会被强制滚动
- 现在：字幕跟随只在播放器区域内滚动，不影响页面其他部分

**筛选条件分享链接打开后不生效**
- 之前：通过带筛选参数的链接进入发现页，筛选条件会被忽略，需要手动重新设置
- 现在：链接中的来源、时间范围、排序等所有筛选参数均正确加载

**手机端播客封面图片被截断**
- 之前：移动端封面固定高度，图片上下被裁切
- 现在：改为正方形自适应，封面完整显示

**创建私有订阅源不再要求 Pro**
- 之前：创建私有订阅源时会错误触发 Pro 身份校验并拒绝请求
- 现在：所有用户均可正常创建私有订阅源

### ⚡ 体验优化

**Pro 早报整体质量提升**
- 优化了个性化内容筛选规则，精选内容与你的兴趣匹配度更高
- 重新设计了播客封面风格，视觉更统一
- 调整了播客脚本生成策略，音频内容更自然流畅

**页面加载速度**
- 文章、视频、播客详情页加载更快，移除了约 450KB 的冗余依赖
- 内容列表页减少不必要的重复渲染，滚动和筛选操作更流畅

## English

### ✨ New Features

**Background audio playback for Daily Brief**
- Audio started in the Brief page continues uninterrupted as you navigate to articles, Explore, or anywhere else
- A mini player bar stays visible at the bottom with progress, pause/resume, and speed controls
- Returning to the Brief page seamlessly restores the full player

**Text/Podcast dual view**
- A new "Text" mode lets you read the brief like an article instead of just listening
- Switching to text mode keeps the audio running in the background — read and listen at the same time

**Public Brief RSS subscription**
- The public daily brief now supports RSS subscriptions — follow in Inoreader, NetNewsWire, or any RSS reader
- Includes full audio metadata so you can listen directly in Overcast, Pocket Casts, or other podcast apps
- Subscribe at `https://bestblogs.dev/en/feeds/rss/daily-brief` (English)

### 🐛 Bug Fixes

**Better empty-state messaging for new Pro subscribers**
- Before: subscribing to Pro after 7:00 AM showed "Your brief is being prepared" — implying it would arrive soon, when it actually wouldn't until the next morning
- Now: first-time subscribers see "Your first brief arrives tomorrow morning"; existing users with no brief today see "Today's brief isn't ready yet" with a link to their most recent one

**Public Brief featured articles showing incomplete info**
- Before: each item only displayed a title — source, type, and score were missing
- Now: full details including source name, content type, and quality score are shown

**Subtitle follow mode hijacking page scroll**
- Before: turning on subtitle follow would force-scroll the entire page while you were browsing the content list below
- Now: subtitle tracking only scrolls within the player panel, leaving the rest of the page undisturbed

**Filter parameters in shared links not applying**
- Before: opening a link with filter parameters (source, time range, sort order, etc.) would silently ignore them
- Now: all URL filter parameters load correctly when the page opens

**Podcast cover image clipped on mobile**
- Before: fixed-height container cropped the cover image top and bottom on mobile devices
- Now: aspect-ratio square sizing keeps the full cover visible at any screen size

**Creating private sources no longer requires Pro**
- Before: creating a private source incorrectly triggered a Pro identity check and rejected the request
- Now: all users can create private sources without restriction

### ⚡ Improvements

**Higher quality Pro Daily Brief**
- Refined content selection rules for better interest matching
- Redesigned podcast cover art style for a more consistent look
- Improved podcast script generation for more natural audio delivery

**Faster page loading**
- Article, video, and podcast detail pages load faster — removed ~450KB of unused dependencies
- Content list renders more efficiently; filtering and scrolling feel smoother
