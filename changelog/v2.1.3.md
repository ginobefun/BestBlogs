## 中文

### ✨ 新功能

**主题解读扩到四种类型**
- 原本的「事件型」「领域型」之外，新增两类：
  - **人物 / 组织型**：聚焦某位个人或团队，含简介、关键贡献、代表作品、观点
  - **对比型**：多个工具 / 方案的横向对比，桌面端是清晰的矩阵表格，移动端是可折叠的维度卡片
- 主题列表顶部按类型筛选，想看哪一类一键过滤
- 重要主题可以置顶（最多 8 条），列表上有 📌 标记

**延伸阅读和引用来源跳站内**
- 主题文末的延伸阅读卡片、底部的引用来源条目，点击后直接进入 BestBlogs 站内文章页
- 阅读路径连贯：主题解读 → 延伸阅读 → 站内文章 → 翻译 / 收藏 / AI 伴读，不再被外链打断
- 如果某篇引用源已下架，会回落显示原文链接（保留可读性）

### 🔧 体验优化

**翻译默认「同篇内容只扣一次」**
- 之前需要手动开启的 dedup 现在默认就是
- 同一篇文章 / 播客 / 视频，跨日期、跨方向（中⇄英）翻译都不重复扣配额
- 修复了一个并发死锁问题，长内容翻译更稳

**后台稳定性**
- 内容抓取和分析链路对外部服务异常更宽容（Dify、RSS 偶发问题不再拖垮整批任务）
- 收紧了告警噪音，关键问题更容易被注意到

## English

### ✨ New Features

**Topic Pages — now four formats**
- Beyond the original Event and Domain formats, two more land:
  - **People / Org**: a person or team in focus — bio, key contributions, signature work, viewpoints
  - **Comparison**: tool / approach matchups — clean matrix table on desktop, collapsible dimension cards on mobile
- A type filter at the top of the topic list lets you jump straight to the one you want
- Editors can now pin up to 8 of the most worthwhile topics to the top of the list (📌 badge)

**Further Reading & References jump in-site**
- Cards at the end of a topic and reference entries at the bottom now open the BestBlogs article page directly
- A continuous reading flow: topic page → further reading → article → translate / bookmark / AI Reader, with no detours through external sites
- If a referenced source has been removed, the original external link is shown as a fallback

### 🔧 Improvements

**Translation: charge once per article, by default**
- The "dedup" toggle that used to require manual enabling is now on out of the box
- The same article / podcast / video doesn't deduct quota again across dates or directions (zh ⇄ en)
- Fixed a concurrency deadlock so long-form translation is steadier

**Backend resilience**
- Content fetch / analysis pipelines tolerate transient external-service hiccups better (Dify, RSS) — fewer batch failures
- Tightened alert noise so the signals that matter actually surface
