## 中文

### ✨ 新功能

**Pro 早报双形态视图**
- Pro 用户的每日早报现在支持「文字版」和「播客版」两种视图，可在早报顶部一键切换
- **文字版**：一句话总结 + 关键洞察列表 + 头条内容卡片，适合扫读和会议间隙快速过一眼
- **播客版**：保持原有的音频 + 封面 + 章节体验，适合通勤和深度收听
- 在 Settings 中可设置默认视图（文字版 / 播客版），打开早报时直接呈现你偏好的形态
- 形态切换即时生效，无需保存

**内建沉浸式翻译 v2**
- 升级为 v2 翻译体系，体验比 v1 更好用、更经济
- **同一篇内容只扣 1 次配额**：跨日期、跨方向（中⇄英）打开同一篇文章 / 播客 / 视频翻译，配额均不重复扣减
- **双向翻译**：同时支持中文 → 英文与英文 → 中文，根据原文语言自动识别
- **全类型覆盖**：从文章扩展到播客转录稿和视频字幕，长文本翻译走专用模型
- **退款窗口**：启动后短时间内主动取消不扣费
- Free 用户保留 3 段预览免费体验，Pro 用户全文无限

**公开站文档中心 `/docs`**
- 上线 BestBlogs 文档中心，包含「快速开始」「功能说明」「FAQ」三大模块
- 内容包括 Daily Brief 介绍、Pro 订阅权益、内容翻译、AI 伴读用法等使用指南
- 支持中英双语，每篇文档独立可分享
- 旧的 `/how-it-works` 页面自动重定向到新文档

**主题深度解读（Topic Pages）**
- 上线由编辑团队精心整理的主题页，包含两类：
  - **事件型主题**：聚焦特定事件（如行业重大新闻），含背景、时间线、各方观点、延伸阅读
  - **领域型主题**：聚焦特定技术 / 业务领域，含核心概念、学习路径、进阶资源、常见问题
- 每个主题页都有相关文章索引，方便顺藤摸瓜
- 中英双语独立内容，搜索引擎友好
- 顶部导航接入「主题」入口

**SEO / AI 引用优化**
- 新增 `/llms.txt` 路由，主动告知 AI 模型 / 搜索引擎 BestBlogs 的内容定位与导航
- 文章和主题页全面接入双语 hreflang 与多语言 alternate 元信息
- 接入站外渠道归因，方便我们了解你是从哪里发现 BestBlogs 的

**AI 伴读（Mobile 内测）**
- 移动端阅读时可向 AI 提问，由 AI 基于当前文章正文做简要回答
- 短回答风格，避免长篇大论；带每次/每日额度防止滥用

### 🔧 体验优化

**邮件投递更稳定**
- 早报、验证码、订阅确认等所有邮件场景，现在底层支持多服务商灾备
- 主投递商出现问题时可秒级切换到备用通道，最大程度保障"每天早上一定能收到早报邮件"
- 邮件链接的打开 / 点击行为统计更准确（之前 click 会被双计为 open）

**搜索体验底层升级**
- 文章 / 订阅源 / 用户兴趣画像底层数据扩充，含双语字段和向量化语义信息
- 为后续上线"语义搜索 / 个性化推荐 v2"打好基础（本次发布尚未启用，待运维灰度）

**Pro 订阅权益完善**
- Pro 自动续费 / 退款 / 切换平台的权益同步更可靠（含日级对账兜底）
- 准备好 iOS App Store 与 Google Play 内购通道（随 Mobile App 正式上架启用）

**Settings 体验**
- Brief 文字版关闭时，顶部 banner 文案改为面向用户的友好表达，不再使用内部研发口吻
- 关闭文字版视图时，你的偏好会被记住，恢复后自动生效

### 🐛 修复

- 邮件链接点击会被错误计为「打开」事件，导致打开率虚高 — 已分离独立计数
- 部分主题列表 / 详情页在暗黑模式下边框 / 文字色对比度不足 — 全面替换为语义色
- 公开站 `/docs` 页面在 iOS Safari / Android Chrome 上被动态地址栏压缩内容 — 改用动态视口高度

### 📱 Mobile App（内测准备）

我们已经把 iOS + Android 移动端的所有核心能力都做完了 — 登录 / 早报 / 内容广场 / 阅读 + 收藏 / AI 伴读 / 推送 / IAP 内购 / 账号注销 — 但本次 v2.1.0 release **不正式上架 App Store / Google Play**，需要先完成 Apple 提审准备和 TestFlight 内测验证。Mobile App 正式发布会另行公告，请关注我们的 X / 微信公众号。

## English

### ✨ New Features

**Dual-View Pro Daily Brief**
- Pro users can now switch between "Text view" and "Podcast view" for the daily brief with a single tap at the top of the page
- **Text view**: one-sentence takeaway + key insights list + headline cards — great for quick skimming between meetings
- **Podcast view**: the original audio + cover + chapters experience — great for commutes and deep listening
- Set your preferred default view in Settings; the brief opens directly in your preferred format
- View switching takes effect instantly, no save needed

**Built-in Immersive Translation v2**
- Translation system upgraded to v2 — friendlier and more economical than v1
- **One charge per resource**: opening the same article / podcast / video translation again — even on a different day or in the opposite direction (zh↔en) — does not deduct quota again
- **Bidirectional translation**: zh→en and en→zh both supported, source language auto-detected
- **Full coverage**: articles, podcast transcripts, and video subtitles all supported; long texts use a dedicated model
- **Refund window**: cancellation shortly after starting does not consume quota
- Free users keep the 3-paragraph preview; Pro users get full unlimited

**Public Docs Center at `/docs`**
- Launched the BestBlogs Docs Center with three sections: Quick Start, Features, and FAQ
- Topics include Daily Brief, Pro subscription benefits, Content Translation, AI Reader usage, and more
- Bilingual content with shareable per-doc URLs
- The old `/how-it-works` page redirects automatically to the new structure

**Topic Pages — Editorial Deep Dives**
- Launched curated topic pages with two formats:
  - **Event topics**: focused on a specific event (e.g., a major industry announcement) — background, timeline, multi-source opinions, related reads
  - **Domain topics**: focused on a technical / business domain — core concepts, learning path, advanced resources, FAQ
- Each topic page links related articles for easy follow-through
- Bilingual independent content with full SEO support
- New "Topics" entry added to the top navigation

**SEO / AI Citation Optimization**
- Added an `/llms.txt` route that actively informs AI models and search engines about BestBlogs' content positioning and navigation
- Full bilingual hreflang and language alternate metadata across articles and topics
- Off-site channel attribution so we can understand where you discovered BestBlogs

**AI Reader on Mobile (closed beta)**
- While reading on mobile, ask questions and get short AI answers grounded in the current article
- Short-form responses with per-minute and per-day quotas to prevent abuse

### 🔧 Improvements

**More Reliable Email Delivery**
- Daily briefs, verification codes, subscription confirmations — every email path now supports multi-provider failover under the hood
- When the primary provider hiccups, traffic switches to a backup provider in seconds — maximizing the "your daily brief always arrives in the morning" guarantee
- Email open / click tracking is now accurate (clicks used to be double-counted as opens)

**Search Infrastructure Upgrade**
- Underlying data for articles / sources / user interest profiles expanded with bilingual fields and vector embeddings
- Lays the groundwork for upcoming semantic search / personalized recommendations v2 (not enabled yet — gradual rollout in operations)

**Pro Subscription Reliability**
- Auto-renewal / refund / cross-platform Pro entitlement sync is more reliable (with daily reconciliation as a safety net)
- iOS App Store and Google Play in-app purchase channels are ready (will activate when Mobile App publicly launches)

**Settings Experience**
- When the Text view kill-switch is on, the banner copy is now user-friendly and no longer leaks internal engineering language
- When text view is unavailable, your preference is remembered and re-applies once the feature is back

### 🐛 Fixes

- Email link clicks were incorrectly counted as "opens", inflating the open rate — clicks now have their own dedicated event
- Some topic list / detail pages had insufficient border / text contrast in dark mode — fully migrated to semantic color tokens
- The `/docs` page on iOS Safari / Android Chrome was being squeezed by the dynamic address bar — switched to dynamic viewport height

### 📱 Mobile App (Closed Beta)

We've completed all core capabilities for iOS + Android — sign-in / brief / explore / reader + bookmarks / AI Reader / push / IAP / account deletion — but **this v2.1.0 release does not publicly ship to the App Store / Google Play**. Apple submission preparation and TestFlight validation come first. The Mobile App's public launch will be announced separately — follow us on X or WeChat for updates.
