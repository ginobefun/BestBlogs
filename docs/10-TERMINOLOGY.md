# BestBlogs 术语与统一表达

更新时间：2026-03-27
状态：Draft

## 1. 文档目标

本文件用于统一 BestBlogs 在产品、设计、文案、架构、运营和对外表达中的关键术语。

目标只有三个：

1. 同一个概念，尽量只保留一个主名称
2. 中文优先自然，英文优先准确
3. 用户看得懂，团队内部也能保持一致

## 2. 使用原则

### 2.1 中文优先自然

中文表达优先选择用户一眼能懂的词，不强行按英文直译。

| 推荐表达 | 不推荐表达 |
|---------|-----------|
| 内容广场 | 发现页 |
| 我的早报 | 个性化早报 |
| 研究报告 | 结构化研究包 |
| 来源偏好 | 来源亲和度 |

### 2.2 动作和关系要分清

| 场景 | 统一用词 | 说明 |
|------|---------|------|
| 用户对内容来源的动作 | 关注 | 不使用「订阅」 |
| 用户对付费方案的关系 | 订阅 Pro / 开通 Pro | 只有付费关系使用「订阅」 |

### 2.3 对外与对内分层表达

用户可见名称优先使用结果导向、体验导向的表达。内部文档和技术文档可以保留更精确的结构术语。

| 用户看到的 | 内部文档可用的 |
|-----------|--------------|
| 研究报告 | 结构化研究包（Context Pack） |
| 公共精选内容 | 由 AI 初评和专家精审共同形成的公共高质量内容 |

### 2.4 首次出现可中英并列

当一个术语需要中英对应时，首次出现可写作「中文（English）」，之后保持统一，不再频繁切换。

## 3. 品牌与定位术语

| 名称 | 中文 | 英文 | 使用场景 |
|------|------|------|---------|
| 品牌母句 | 让高质量内容找到对的人 | Help high-quality content find the right people | 愿景、品牌总述 |
| 用户主 slogan | 发现真正适合你的高质量内容 | Discover high-quality content that truly fits you | 首页、OG、海报、SEO H1 |
| 官方一句话定位 | AI 驱动的个性化高质量阅读工作流 | AI-powered personalized high-quality reading workflow | 官网、媒体资料、产品简介 |
| 中文自然版定位 | 帮你持续发现高质量内容的个性化阅读产品 | A personalized reading product for discovering high-quality content | 中文产品介绍、社交媒体简介 |
| 长期定义 | 高质量阅读与知识基础设施 | High-quality reading and knowledge infrastructure | 长期方向说明，不作为首页主文案 |

## 4. 产品骨架术语

| 中文 | 英文 | 定义 | 使用层级 | 备注 |
|------|------|------|---------|------|
| 公共精选内容 | Public Curated Content | 平台面向所有用户提供的公共高质量内容集合，当前以技术内容为主 | 对外主用、产品文档主用 | 由 AI 初评和专家精审共同形成。战略文档中如需强调机制，可进一步补充说明 |
| 个人阅读工作流 | Personal Reading Workflow | 围绕我的早报、为你推荐、AI 伴读、我的关注形成的持续阅读体验，也就是让内容越来越适合你的日常阅读流程 | 产品文档主用 | 用户界面中优先呈现具体入口名称 |
| 可调用基础设施 | Agent-ready Infrastructure | 为智能体调用、理解和复用内容与上下文而准备的基础设施层，支持用户将知识相关原语灵活组合与编排 | 内部主用、架构主用 | 长期方向，不等同于通用智能体容器 |
| 每日节奏 | Daily Rhythm | 围绕发现、阅读、回顾与沉淀形成的连续体验节奏，强调稳定积累而非机械重复任务 | 对外主用、产品文档主用 | 中文统一使用「每日节奏」，英文统一使用 Daily Rhythm |

## 5. 用户可见核心术语

### 5.1 界面与功能

| 中文 | 英文 | 定义 | 使用层级 | 备注 |
|------|------|------|---------|------|
| 内容广场 | Explore | 面向公共高质量内容的开放浏览入口 | 对外主用、产品文档主用 | 替代「发现页」。UI 和路由中可保留 Explore |
| 我的早报 | My Brief | 基于用户关注、兴趣和阅读行为生成的个人化早报 | 对外主用、产品文档主用 | 所有文档和界面统一使用，不再使用「个性化早报」「用户早报」 |
| 公共早报 | Public Brief | 面向所有用户的公共早报，由公共高质量内容整理而成 | 对外主用、产品文档主用 | 与「我的早报」配对使用 |
| 为你推荐 | For You | 围绕用户兴趣和行为生成的持续推荐内容流 | 对外主用、产品文档主用 | 界面名称可保留 For You |
| AI 伴读 | AI Reading Assistant | 帮助用户更快理解内容的阅读辅助能力，包括提问、总结、章节跳转等 | 产品文档主用、对外主用 | |
| 我的关注 | Following | 用户关注的订阅源所形成的内容入口或内容流 | 对外主用、产品文档主用 | 替代「订阅流」 |
| 每日回顾 | Daily Review | Pro 专属的晚间回顾能力，21:00 基于当日阅读行为生成 | 对外主用、产品文档主用 | 与「每日早报 Daily Brief」配对形成早晚闭环；v2 升级为夜间叙事页 |
| 夜间叙事页 | Evening Narrative Page | Daily Review v2 的页面定位：从「模块拼贴」升级为具备主题聚合与情感开场的叙事式体验 | 产品文档主用 | 见 `specs/daily-review-v2-redesign.md`；文案不得使用装饰性 emoji 或感叹号 |
| 主题聚合 | Topic Clustering | Dify workflow 内部 LLM 软聚类，把当日阅读聚到 2–3 个主题并产出一句话解读 | 产品文档主用、技术文档主用 | 不依赖后端向量计算；主题命名避免「其他」「综合」「杂项」等宽泛词 |
| 今日一问 | Daily Question | 基于当日阅读主题由 AI 生成的一句开放式思考题，替代 v1 的 thoughtPrompt | 对外主用、产品文档主用 | v2 MVP 为内联输入框（不触发 Copilot）；禁用感叹号和 emoji |
| 周回顾 | Weekly Review | Daily Review 的周度聚合视图，每周日产出本周主题图谱与未解问题 | 产品文档主用 | Phase 2 候选能力，MVP 不做 |

### 5.2 用户动作

| 中文 | 英文 | 定义 | 使用层级 | 备注 |
|------|------|------|---------|------|
| 收藏 | Bookmark | 将内容保存下来，方便后续回看 | 对外主用、产品文档主用 | |
| 划线 | Highlight | 在阅读中标记关键片段 | 对外主用、产品文档主用 | |
| 不感兴趣 | Not Interested | 用户对内容的负反馈操作 | 对外主用、产品文档主用 | 文档中可写作「标记为不感兴趣」 |
| 分享 | Share | 用户将内容分享到外部平台或生成分享链接 | 对外主用、产品文档主用 | |
| 稍后读 | Read Later | 用户标记内容为稍后阅读 | 对外主用、产品文档主用 | 常见于早报和推荐流场景 |
| 标记已读 | Mark as Read | 用户将内容标记为已阅读 | 对外主用、产品文档主用 | |

### 5.3 体验与指标概念

| 中文 | 英文 | 定义 | 使用层级 | 备注 |
|------|------|------|---------|------|
| 深度阅读 | Deep Read | 从早报或为你推荐点入详情页后，阅读时长超过阈值或触发 AI 伴读交互 | 产品文档主用、数据分析主用 | 产品指标关键概念，具体阈值见 CURRENT_STATE.md |

## 6. 内容类型术语

| 中文 | 英文 | 定义 | 备注 |
|------|------|------|------|
| 文章 | Article | 来自博客、媒体或个人网站的文本内容 | 不使用「博客文章」「博文」 |
| 播客 | Podcast | 音频形态的内容，通常来自播客频道的单集节目 | 界面中可保留 Podcast |
| 视频 | Video | 视频形态的内容，来源包括 YouTube、Bilibili 等平台 | 不区分平台。如需指明来源，写作「来自 YouTube 的视频」 |
| 推文 | Tweet | 来自 X 的短内容 | 标注来源时使用 X |
| 期刊 | Newsletter | 通过邮件订阅的定期内容 | 技术文档中可使用 Newsletter |

## 7. AI 能力与内容审核术语

| 中文 | 英文 | 定义 | 备注 |
|------|------|------|------|
| AI 初评 | AI Screening | AI 对新采集内容进行结构化评分和质量判断的过程 | 不再使用「AI 评分」「AI 筛选」「AI 过滤」 |
| 专家精审 | Expert Review | 人工审核员对 AI 初评结果进行复核和最终判断的过程 | 不再使用「人工审核」「人工复审」「人工精审」 |
| AI 摘要 | AI Summary | AI 为内容生成的结构化摘要，帮助用户快速了解要点 | |
| AI 问答 | AI Q&A | AI 伴读中的互动问答能力，用户可以针对内容提问 | |
| 章节跳转 | Section Jump | AI 伴读中帮助用户快速定位到内容特定章节的能力 | |
| 内建翻译 / 沉浸式翻译 | Content Translate / Immersive Translate | Pro 权益之一：英文内容段落级双语对照翻译，替换外部 wenrun.ai 跳转 | 对外可用「沉浸式翻译」，对内代码统一 `content-translate` |

## 8. 订阅源与用户关系术语

这是最容易混淆的一组，需特别统一。

| 中文 | 英文 | 定义 | 备注 |
|------|------|------|------|
| 订阅源 | Source | 可以持续提供内容更新的来源对象，如博客、RSS、播客、视频频道等 | 不使用 Subscribe Source |
| 关注 | Follow | 用户将某个订阅源加入自己的持续关注列表 | 用户动作统一使用「关注」 |
| 取消关注 | Unfollow | 用户将某个订阅源移出自己的持续关注列表 | |
| 私有订阅源 | Private Source | 由用户自己添加、仅自己使用的订阅源 | 替代「个人订阅源」「个人来源」 |
| 公共订阅源 | Public Source | 由平台维护、可供所有用户浏览和关注的订阅源 | |
| 我的关注 | Following | 用户关注后的订阅源内容集合 | 用户侧入口名称，不是内部数据结构名 |
| 添加新来源 | Add Sources | 从公共订阅源中挑选扩充到我的关注；对应路由 `/me/sources/add`，与 `/me/sources`（管理态）分列 | 用户界面使用「添加新来源」，不使用「全部订阅源」「所有订阅源」 |
| 特别关注 | Featured | 用户对某个订阅源标记的重点偏好（星标），在个性化早报生成时获得更高权重 | `UserSubscriptionEntity.featured` 字段 |
| 智能视图 | Smart View | 我的关注 sidebar 中的派生切片节点，包括「特别关注 / 未分类 / 最近更新 / 长期未更新」 | 派生自现有字段（`featured` / `folderId` / `lastLiveTime`），非用户创建的分组 |
| 关注贡献 | Source Contribution | 某个已关注的订阅源在近 N 天内贡献到用户个性化早报的条目数，反映「我的关注 → 我的早报」的因果反馈 | 对应 API `/api/users/me/subscriptions/contributions?days=N`，卡片上显示「近 N 天进入早报 X 次」 |
| 来源偏好 | Source Preference | 系统基于用户关注和行为形成的对不同订阅源的偏好判断 | 替代「来源亲和度」。代码中可保留 `source_affinity` |

## 14. 维护规则

### 新增术语的标准格式

每个术语条目建议包含五列：中文主名称、英文对应、一句话定义、使用层级、备注（容易混淆时必须填写）。

### 更新原则

1. 新增功能或概念时，同步在本文件中添加对应术语
2. 术语变更时，同步更新第 12 节（废弃表达）和第 13 节（速查表）
3. 若新术语与已有术语存在语义重叠，优先合并而非新增
4. 与 CLAUDE.md 中「术语不一致：更新 10-TERMINOLOGY.md」的维护规则保持一致
