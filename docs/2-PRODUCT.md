# BestBlogs 产品蓝图

更新时间：2026-03-27
状态：Released

## 1. 产品定义

BestBlogs 是 AI 驱动的个性化高质量阅读工作流。

它不是内容聚合器，不是追热点的推荐流，而是围绕高质量内容的发现、判断、理解与沉淀构建的产品系统。当前以阅读工作流为主要形态，长期延展为面向阅读者、创作者和智能体的高质量阅读与知识基础设施。

**一句话定位：** 公共策展建立信任，个人工作流驱动留存，可调用基础设施打开天花板。

## 2. 我们服务谁

BestBlogs 面向需要高质量输入、理解与表达的人。

当前最清晰的切入人群是信息密集型人群，包括技术从业者、独立开发者、产品人、管理者和研究者。他们共同的特点是需要持续获取高质量信息来支撑工作、学习、判断和输出。

公共策展以技术内容为起点建立质量品牌，个人层天然不受领域限制，可以覆盖更广泛的主题、来源与兴趣范围。长期来看，BestBlogs 既服务阅读者，也逐步服务创作者，并最终支持智能体使用这些内容与上下文能力。

## 3. 我们解决什么问题

### 3.1 发现与判断的鸿沟

真正稀缺的不是内容，而是判断什么值得优先看、什么真正与自己相关。多数产品要么只是搬运内容，要么把相似的热门内容推给所有人，缺少稳定的质量判断与优先级排序。

### 3.2 吸收效率的瓶颈

长内容多，理解成本高。用户需要先快速掌握重点，再决定是否深入阅读。没有压缩和优先级排序能力的产品，实际上在浪费用户最稀缺的资源，也就是注意力。

### 3.3 阅读无法沉淀为知识

多数阅读行为停留在看过的层面，难以形成回顾、积累与复用。碎片化阅读缺少持续跟踪与沉淀机制，读过的好内容最终被遗忘。

### 4.2 结构性差异

## 5. 产品飞轮：三层递进模型

飞轮是 BestBlogs 产品结构的核心。三层不是三个独立产品，而是同一个飞轮的三次加速。

### 5.1 第一层：公共策展质量池

面向所有用户的公共层。核心任务是通过 AI 初评加专家精审的双层机制，把高噪声信息压缩为可信赖的内容集合。

**承载能力：**
- 内容采集、分析、筛选与审核的完整链路
- 公共早报，覆盖文本、音频和邮件三种形态
- 精选内容与公共分发入口
- SEO 与品牌心智建设

**战略角色：** 证明 BestBlogs 具备真实的内容判断力。这是获客入口，也是信任基础。质量池是随时间复利增长的资产，每一次评审都在训练更精准的判断标准。

**当前状态：** █████████░ 成熟，内容全链路完整运行，公共早报已上线，AI 初评加专家精审闭环已验证。

### 5.2 第二层：个人 AI 阅读工作流

面向用户个体的阅读工作流层，也是 Pro 的核心价值承载层。这一层不再只是替大家选什么，而是替你选什么、让内容更适合你。

**承载能力：**
- 我的早报与为你推荐
- 六维兴趣标签与行为学习
- 订阅源管理，包括公共和私有两种形态
- AI 伴读，支持摘要、互动问答和章节跳转
- 回顾与沉淀能力，方向已确定但尚未完整构建

**战略角色：** 把公共判断力转化为面向单个用户的持续工作流。这是留存和付费的核心驱动力。扩展到更多领域时，优先通过第二层的自定义与个性化能力实现，而不是直接稀释第一层的公共策展边界。

**当前状态：** ██████░░░░ 建设中，我的早报、为你推荐、AI 伴读和订阅源管理已上线或接近上线。六维兴趣标签系统设计完成，待实施。回顾和沉淀尚在规划中。

### 5.3 第三层：可调用知识基础设施

BestBlogs 不只服务界面中的用户，也逐步服务外部系统和智能体。

**承载能力（长期方向）：**
- 结构化研究包，将高质量内容打包为可直接消费的研究上下文
- 主题监控规则，支持对特定主题的持续追踪与通知
- 早报原语，让外部系统可以调用早报的生成能力
- 研究会话，提供面向深度研究场景的交互式接口
- MCP 服务，面向智能体的标准化工具接口

**战略角色：** 让高质量内容和判断力成为人和智能体都可以调用的基础设施。BestBlogs 不做通用智能体入口，而是聚焦成为智能体时代默认可调用的阅读与知识基础设施。

**当前状态：** ██░░░░░░░░ 方向性，OpenAPI 2.0 已上线，基础鉴权链路已通。智能体架构设计已批准，待实施。

## 7. 核心产品工作流：每日节奏

BestBlogs 的核心不是某个单独功能，而是一个完整的每日节奏。

```
    ┌──────────┐      ┌──────────┐      ┌──────────┐      ┌──────────┐
    │  发现     │─────►│  阅读     │─────►│ 回顾/沉淀 │─────►│ 创作/分享 │
    └──────────┘      └──────────┘      └──────────┘      └──────────┘
     ██████████        ██████████        ████░░░░░░        ██░░░░░░░░
     当前成熟           当前成熟           建设中             方向性

     公共早报        内容详情页          （规划中）          （长期方向）
     我的早报        AI 伴读            每日回顾            知识复用
     为你推荐           划线提问            研究会话            创作辅助
     搜索发现          摘要 + 章节跳转      深度笔记            分享
```

**当前重点：** 把发现和阅读两段做到极致，让回顾和沉淀从阅读工作流自然生长出来。创作和分享是每日节奏的长期终点，当前不进入主线。

### 7.1 发现

帮助用户知道今天什么值得优先看。由公共早报、我的早报、为你推荐和搜索发现共同承担。不是把更多内容摆到用户面前，而是降低该看什么的判断成本。

### 7.2 阅读

帮助用户更高效地理解内容。包括内容详情页、AI 伴读、划线提问、摘要与章节跳转等能力。把阅读从被动消费转向更主动的理解过程。

### 7.3 回顾与沉淀

帮助用户留下真正重要的东西。包括当天内容复盘、重点整理、后续研究和结构化笔记。这些能力从阅读工作流自然生长而来，不是独立产品。

### 7.4 创作与分享

每日节奏的长期终点。当回顾与沉淀稳定后，用户不仅消费内容，而是组织观点、形成输出、复用上下文。当前阶段视为从回顾和沉淀自然延伸的长期方向。

## 10. 成功指标框架

### 10.1 北极星指标：深度阅读闭环率

周活跃阅读用户中，完成从发现到深度阅读闭环的比例。

深度阅读的操作定义是，从早报或为你推荐点入内容详情页后，阅读时长超过阈值或触发 AI 伴读交互。具体阈值在 CURRENT_STATE.md 中维护。

这个指标综合反映了三个层面：内容质量，即用户愿意从早报或为你推荐点进去；个性化精度，即推荐的内容确实值得读；阅读体验，即用户真的读完了，而不只是扫一眼标题。

### 10.2 分层健康指标

| 层级 | 指标类型 | 关注什么 |
|------|---------|---------|
| **第一层** | 内容质量 | 公共早报点击率、精选内容的阅读完成率、策展准确率（专家精审通过率） |
| **第一层** | 获客效率 | SEO 自然流量、早报邮件打开率、新用户注册转化率 |
| **第二层** | 个性化精度 | 我的早报与公共早报的点击率差值、为你推荐点击率、兴趣标签覆盖率 |
| **第二层** | 工作流粘性 | 7 日和 30 日留存、日均阅读篇数、AI 伴读使用率 |
| **第二层** | 付费价值 | Free 到 Pro 转化率、Pro 续订率、Pro 用户满意度 |
| **第三层** | 可调用性 | API 调用量、外部集成数、智能体使用频率 |

### 10.3 反指标，我们拒绝优化的方向

- **阅读时长。** 我们帮用户读得更少而非更多。时长增加可能意味着内容质量下降或推荐不精准。
- **内容数量。** 更多不是更好。质量池的扩大必须以判断力为前提。
- **无深度的日活数。** 没有阅读深度的日活用户数是空洞的。

## 11. 阶段演进与阶段门槛

### 11.1 阶段全景

与愿景文档一致，BestBlogs 的演进分为两个阶段。第二阶段的两个方向，即知识沉淀与智能体基础设施，是并行延伸，不是先后关系。

```
  第一阶段                    第二阶段（两个方向并行延伸）
  个性化阅读工作流             ┌─ 向上：知识沉淀能力
  ═══════════════            │  ═══════════════════
  ◄── 当前阶段 ──►           │  验证：回顾和沉淀能否从阅读自然生长
                             │  关键产出：
                             │  · 每日回顾 · 研究会话
  验证：                      │  · 深度笔记 · 知识图谱 · 阅读回顾
  个性化工作流能否             │
  稳定带来留存与付费           ├─ 向外：可调用基础设施
                             │  ═══════════════════
  关键产出：                   │  验证：智能体调用是否产生独立价值
  · Pro 订阅闭环              │  关键产出：
  · 我的早报                │  · 结构化研究包 · 主题监控规则
  · 为你推荐 · AI 伴读         │  · MCP 服务 · 早报原语
  · 六维兴趣标签              │  · 面向智能体的工具接口
                             └─────────────────────────
```

### 11.2 阶段门槛

阶段转换不是日历驱动，而是信号驱动。以下定义的是信号类型和判断逻辑，具体数值目标在 CURRENT_STATE.md 中维护。

**第一阶段到第二阶段的门槛：**

| 信号类型 | 判断逻辑 |
|---------|---------|
| 留存验证 | Pro 用户 30 日留存稳定在可接受水平，且留存主要由工作流使用驱动而非新鲜感 |
| 付费验证 | Free 到 Pro 转化率稳定，且 Pro 续订率表明用户认可持续价值 |
| 工作流闭环 | 用户日常使用路径稳定覆盖发现到阅读的链路，早报和为你推荐成为主要入口 |
| 质量池规模 | 公共质量池日均新增内容量与质量判断准确率达到自运转水平 |

**进入第二阶段后，两个方向各自的推进信号：**

| 方向 | 信号类型 | 判断逻辑 |
|------|---------|---------|
| 向上：知识沉淀 | 沉淀行为 | 用户的回顾和沉淀行为自然发生，不需要强引导 |
| 向上：知识沉淀 | 知识复用 | 沉淀的内容被用户实际回顾和引用，而非仅仅收藏 |
| 向外：可调用基础设施 | 外部需求 | 开始收到来自开发者或智能体的 API 调用需求 |
| 向外：可调用基础设施 | 数据准备 | 结构化分析和用户上下文数据足以支撑有价值的 API 输出 |

## 12. 增长与商业逻辑

增长与商业化内嵌在飞轮结构中。

```
  获客 ◄──── 第一层公共策展
    │         公共早报 · 精选内容 · SEO
    ▼
  激活 ◄──── 首次我的早报预览
    │         用户感受到 BestBlogs 真的开始理解我关注什么
    ▼
  留存 ◄──── 第二层个性化工作流
    │        早报越来越懂你，为你推荐越来越准
    ▼
  付费 ◄──── Pro 价值 = 内容越来越贴合
    ▼
  扩展 ◄──── 第三层可调用基础设施
```

**激活关键：** 让用户在新用户引导阶段尽快感受到个性化价值，而不是展示功能列表。

**留存关键：** 工作流越用越适合，形成迁移壁垒。当我的早报、为你推荐、AI 伴读协同运转时，BestBlogs 成为用户每天高质量输入的一部分。

## 13. 战略风险与应对

| 风险 | 应对策略 |
|------|---------|
| **大模型成本飙升** | 多模型策略分级调用；缓存层减少重复调用；关键路径成本监控 |
| **内容源封锁** | 多源冗余；用户私有订阅补充公共源；与创作者建立合作关系 |
| **竞品复制策展模式** | 质量池是时间复利资产，先发优势显著；持续投入判断力而非内容量 |
| **单人团队瓶颈** | AI 工具最大化个人产出；自动化运营流程；优先级严格按飞轮核心排序 |
| **冷启动与初始流量** | 技术社区精准切入；内容质量本身是最好的获客方式 |

## 14. 产品边界：我们不做什么

- **不以时间消耗为目标。** 永远不追求让用户刷更久。帮用户更快获取价值，然后把时间还给他们。
- **不做泛化 AI 工具箱。** AI 能力始终围绕阅读与知识场景，不发散为通用对话、通用写作或通用搜索工具。
- **不做通用智能体入口。** 聚焦成为可被智能体调用的阅读与知识基础设施，而不是智能体的调度中心。
- **不以量取胜。** 不以更多内容作为核心竞争方向。更适合、更高质量、更可用，永远优先于更多。
