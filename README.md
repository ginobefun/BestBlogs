## 1. 为什么 BestBlogs

如果你也是「订阅了 200 个 RSS 源，但每天打开 reader 都想关掉」那种状态，下面这三种疲劳你一定不陌生：

| 自管理 RSS 阅读器的三大疲劳 | BestBlogs 的回应 |
|---|---|
| **信息过载**：每天 200+ 篇，根本读不完 | AI 在源头做**初筛 + 六维评分**，每天只把高分内容浮上来 |
| **找不到精华**：标题党、转载、机翻、营销稿混在一起 | AI 初评 + 编辑精审形成「公共质量池」，所有用户共享这份判断力 |
| **没有评分**：不知道哪些值得深读 | 每篇内容都有**选题 / 内容 / 深度 / 实用 / 创新 / 表达**六维评分，深读价值一目了然 |
| **跨语言断层**：英文一手内容看不动，等中文转述又慢又失真 | 中英双向沉浸式翻译 + AI 伴读，让你直接读原文 |
| **重复阅读**：同一条新闻在 5 个源里被洗了 5 遍 | 主题解读把碎片聚合成主题全景，事件 / 领域 / 人物 / 产品对比四类视角 |

> **BestBlogs 的承诺不是「更多内容」，而是「更适合你的高质量内容」。**

## 2. 是什么

BestBlogs.dev 是 **AI 驱动的私人阅读助手**。产品结构上分两端：

- **公共策展层** —— 让所有访问者看到 BestBlogs 当下精选的高质量内容（无需登录）
- **我的空间** —— 让登录与 Pro 用户拥有真正属于自己的阅读流

两端由同一个公共质量池（AI 初评 + 编辑精审）供能。

### 核心能力

**公共策展层（任何人无需登录可读）**

| 入口 | 说明 |
|---|---|
| **每日早报** | 中英双语图文 + 10-15 分钟播客（Apple / Spotify / 小宇宙）+ 海报 / Telegram / X / RSS 多渠道分发 |
| **精选周刊** | 每周精审一份高质量内容合集，对本周值得回顾的内容做二次提纯 |
| **主题解读** | 围绕重要事件 / 领域 / 人物 / 关键产品对比形成的编辑式深度解读，把碎片内容聚合成主题全景 |
| **内容广场** | BestBlogs 全部公开内容的浏览主入口，「精选 / 最新」双视角，不做个性化排序 |

**我的空间（登录可见，Pro 解锁完整工作流）**

| 入口 | 说明 |
|---|---|
| **我的早报** | 基于关注、兴趣标签与阅读行为的个性化每日早报（图文 + 邮件），Pro 解锁；**Pro 开通后 15 分钟内即送首份** |
| **我的关注** | 五种源（RSS / Newsletter / Twitter / YouTube / Podcast）三列工作台统一管理；粘贴 YouTube / 小宇宙 / 公众号链接即可自动识别并接入 |
| **我的阅读** | AI 伴读协助理解（摘要 / 翻译 / 提问 / 章节跳转）+ 图书馆沉淀（书签 / 划线 / 历史） |
| **我的回顾** | 每天晚间自动生成一份回顾页，把当日阅读足迹组织成「一天的阅读小结」 |

**跨入口能力**

| 能力 | 说明 |
|---|---|
| **AI 内容分析** | 六维评分（选题 / 内容 / 深度 / 实用 / 创新 / 表达）+ 摘要 + 关键观点 + 金句 |
| **沉浸式翻译** | 中英双向 + 文章 / 推文 / 播客 / 视频全类型支持 |
| **可解释画像** | 兴趣偏好由显式行为驱动（关注 / 阅读 / Domain 自定义篇数），不开放手动调权 |

## 4. 快速上手

**直接浏览**：访问 [bestblogs.dev](https://bestblogs.dev/?utm_source=github&utm_medium=readme&utm_campaign=quickstart-web)，免费阅读所有公开内容。


## 7. 开放 API（v2）

BestBlogs v2 API 面向开发者开放，提供内容发现、阅读与个人数据管理能力。

**Base URL**：`https://api.bestblogs.dev/openapi/v2`  
**鉴权**：Header `X-API-KEY: <your_key>`（在 [Settings](https://bestblogs.dev/settings?utm_source=github&utm_medium=readme&utm_campaign=openapi) 生成）

| 模块 | 说明 |
|---|---|
| [openapi/01-auth.md](./openapi/01-auth.md) | 认证与身份，公共规范 |
| [openapi/02-intake.md](./openapi/02-intake.md) | 兴趣画像建立，冷启动引导 |
| [openapi/03-discover.md](./openapi/03-discover.md) | 内容发现：早报 / 推荐 / 搜索 |
| [openapi/04-read.md](./openapi/04-read.md) | 内容阅读：全文 / Markdown / 元数据 |
| [openapi/05-capture.md](./openapi/05-capture.md) | 书签 / 划线笔记 / 阅读历史 |
| 主题（Topics） | `GET /openapi/v2/topics`（列表） + `GET /openapi/v2/topics/{slug}`（详情，含四类型结构化字段、引用、延伸阅读）— 鉴权同上，独立文档整理中 |

## 8. CLI & Agent Skills

### @bestblogs/cli

官方命令行工具，基于 OpenAPI v2，所有命令支持 `--json` 模式，可直接被 AI Agent 消费。

```bash
npm install -g @bestblogs/cli
bestblogs auth login
bestblogs intake setup           # 冷启动：选兴趣标签
bestblogs discover today --limit 20
bestblogs read deep <resourceId>
bestblogs capture bookmark add <resourceId> --note "值得反复读"
```

- 文档：[cli/README.md](./cli/README.md)
- 更新日志：[cli/CHANGELOG.md](./cli/CHANGELOG.md)
- 源码：[cli/src/](./cli/src/)（TypeScript，MIT 协议）

### BestBlogs Skills

一组 SKILL.md 文件，让 Claude Code / Codex / Cursor / OpenClaw 等 Agent 主动触发 BestBlogs 能力（25 个稳定原语）。

```bash
npx @bestblogs/skills             # 默认安装到 Claude Code 和 Codex
npx @bestblogs/skills --client codex
npx @bestblogs/skills upgrade     # 升级到最新版
```

安装后直接说：
- 「今天有什么值得读的？」→ `bestblogs-discover`
- 「深度阅读这篇」→ `bestblogs-read`
- 「收藏这篇，加笔记」→ `bestblogs-capture`
- 「为什么推这条给我？」→ `bestblogs-explain`
- 「BestBlogs 有哪些主题页 / 看 X 主题」→ `bestblogs-topic`

- 文档：[skills/README.md](./skills/README.md)

## 11. 实现原理

### AI 内容处理流水线

```
RSS 爬取 → 初筛过滤 → AI 深度分析 → 多语言翻译 → 入库 → 个性化推荐
```

**1. 内容爬取**：基于 RSS + 无头浏览器提取全文，支持订阅源配置正文选择器

**2. 初筛过滤**：语言类型、内容质量特征初步评分，过滤低价值内容

**3. AI 深度分析**：大语言模型生成六维评分 + 摘要 + 关键观点 + 金句 + 标签

**4. 多语言翻译**：识别专业术语 → 初译 → 检查 → 意译优化，支持中英双语

**5. 个性化推荐**：六维兴趣标签匹配 + 早报智能编排
